# Epic 1 快速人工验证（Open WebUI）

版本: v1.2
目标: 用最少步骤确认 Epic 1 核心链路可用

## 1) 先访问 Open WebUI

- 如果你本机启动了 Open WebUI，通常访问 `http://localhost:3000`
- 如果是远端/容器部署，访问你实际配置的 Open WebUI 地址
- 登录后进入模型连接配置页（Connections / Models）

## 2) 配置 Agent 端点

- Base URL 填: `http://agent-service:8100/v1`（容器内网络）
- API Key 填: 你的 `AGENT_API_KEY`
- 保存后选择这个连接对应的模型

备注: 如果 Open WebUI 不是和 `agent-service` 在同一网络，Base URL 需要改成实际可达地址（例如宿主机 IP + `8100`）。

## 3) 10 分钟验证步骤

### Step 1: 基础对话

- 发送: `你好，简单介绍你自己`
- 通过标准: 能正常流式回复，无报错

### Step 2: Mentor Persona 行为

- 发送: `什么是闭包？`
- 通过标准: Agent 先反问前置知识（如"你了解作用域/函数吗？"），而非直接解释闭包定义。回复风格为引导式提问。

### Step 3: Tool Loop

- 发送: `请调用 echo 工具返回 hello tool loop`
- 通过标准: 能看到处理中状态（Thinking/Running 类文本）+ 最终结果包含 `hello tool loop`
- 备注: 模型可能自行判断不需要调用工具而直接回复，这也算正常行为。关键是如果调用了工具，状态事件和结果能正确展示。

### Step 4: 连续可用性

- 发送: `继续解释上一步你做了什么`
- 通过标准: 会话不中断，可连续回复

### Step 5 (可选): 失败兜底

- 在测试环境模拟上游不可用后，再发送任意问题
- 通过标准: 返回可理解错误，不崩溃
